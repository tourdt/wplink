# 项目规则

1. 先想再写：编码前梳理逻辑、需求假设，有歧义主动确认，理清方案再动笔～
2. 简洁优先：用最少代码完成需求，不搞多余功能和冗余抽象，拒绝复杂写法哦
3. 精准改动：只改和需求直接相关的代码，不擅自动无关部分，改动范围贴合目标
4. 目标驱动：全程围绕业务核心目标写代码，明确验证标准，不偏离需求核心啦
5. 服务器开发：尽量按照 go-zero 的开发规范进行开发；涉及 `.api`、handler、types、svc、model 等能由 goctl 自动生成的代码，优先使用 goctl 生成，不手写可生成代码；数据库表对应的 `model` 代码应通过 goctl model 生成，并以表结构或 schema 作为来源。
6. 文档语言：以后所有实施计划、设计文档、产品文档必须使用中文撰写；涉及代码标识、文件路径、接口名、命令等可保留英文原文。
