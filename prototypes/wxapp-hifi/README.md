# 小程序高保真流程原型

入口文件：`index.html`

验证目标：

- 用户能否快速理解平台解决的是找货、找厂、清库存。
- 买家能否从首页搜索进入资源详情并联系商家。
- 搜索无结果时，用户能否自然转为提交采购需求。
- 商家能否理解发布资源、认证权益和发布效果。
- 商家能否查看自己发布过的资源，并执行刷新、置顶、标记成交、再发类似资源。
- 消息中心能否承接审核、过期、撮合和效果反馈。
- 推广位是否能以“产业资源推广/运营推荐”的方式出现，而不是让用户感到是泛广告。
- 首页 Banner 是否能配置多条并以横滑列表展示。
- 每条 Banner 是否能配置不同跳转目标，如专题、活动网页、商家主页。
- Banner 是否支持配置活动推广页，并通过 URL 打开小程序 web-view。

本版新增推广位验证：

- 首页本周重点运营位。
- 首页平台推荐资源位。
- 搜索结果置顶提示。
- 资源详情同类推荐。
- 商家主页权益位。
- 我的页置顶券权益提醒。
- 我的发布/资源管理页。
- 首页 Banner 专题列表页。
- 活动推广 web-view 模拟页。

当前原型为静态交互稿，不连接真实后端接口。确认流程后，再拆分为 uni-app 页面。
