# wxapp 发布页重新设计

## 目标

发布页按“快速发布 + 提升填写质量”优化，保持商家填单效率，同时用清晰分区、完成度提示和图片预览减少漏填、乱填。

本次不参考原型图，不修改接口协议、路由、发布流程、草稿流程和上传流程。

## 页面结构

- 基础信息：紧凑完成度提示、资源类型、标题、品类。
- 供应信息：数量/产能、价格描述、资源描述。
- 资源图片：参考商家资料页的 3 列图片网格，最后一格添加，图片格支持预览和删除。
- 联系信息：联系人、联系电话。
- 底部固定操作：参考商家资料编辑页，使用全宽白色固定操作栏，包含保存草稿和提交审核。

## 交互规则

- 必填项仍为商家 ID、资源类型、标题、品类、联系人、联系电话。
- 完成度仅用于前端提示，放在基础信息顶部，不改变原有校验逻辑。
- 图片上传成功后展示在网格中；点击图片预览，点击右上角删除移除当前图片 URL。
- 发布页加载商户资料后，联系人姓名默认使用商户资料中的 `contact.name`；联系电话只有接口返回完整 `contact.phone` 时才默认填入，不使用脱敏手机号。
- 草稿和提交按钮保留原有函数与接口调用。

## 验证标准

- 发布页源码包含新分区结构和固定底部操作。
- 现有 `validate:pages` 和流程测试通过。
- 新增发布页结构测试通过。
- 页面不引入新的接口依赖，不影响已保存的商家 ID 和路由参数。
