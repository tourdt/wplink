# wxapp 商家主页布局优化设计

## 目标

优化 `wxapp/pages/merchant/detail.vue` 的商家主页布局，让买家进入页面后优先判断商家身份、可信度和主营能力，再查看实拍图片、地址和发布资源。

本次采用已确认的 A 方案“信任首屏”：把商家身份、认证标签、主营摘要和发布统计集中到首屏头部，减少分散卡片带来的阅读成本。

## 范围

本次覆盖：

- `wxapp/pages/merchant/detail.vue` 的模板结构调整。
- `wxapp/pages/merchant/detail.vue` 的局部样式优化。
- 现有商家主页字段的展示顺序和兜底文案。

本次不覆盖：

- 后端 API、数据库字段或接口协议调整。
- `ResourceCard` 组件重构。
- 商家资料编辑页 `wxapp/pages/merchant/profile.vue`。
- 关注、编辑、地图导航、资源跳转和底部联系提示的业务逻辑变更。

## 页面结构

### 信任头部

顶部使用深色商家身份卡，集中展示：

- 商家 LOGO；无 LOGO 时展示商家名称首字占位。
- 商家名称。
- 主营摘要，优先使用主营品类，其次使用简介，最后兜底为“服装产业资源商家”。
- 商家类型标签。
- 已认证标签。
- 平台核实标签。
- 关注按钮；当前登录商家查看自己的主页时展示“编辑”按钮。

头部卡下半部分直接展示 3 个统计项：

- 当前资源。
- 历史发布。
- 成交反馈。

统计仍使用现有 `resourcesSummary` 和 `merchantResources` 兜底，不新增接口字段。

### 商家画像

头部之后前置一个“商家画像”内容块，合并原本分散的主营品类、信用标签和商家简介：

- 主营品类以标签形式展示；缺失时展示“主营品类待补充”。
- 信用标签紧跟主营品类展示；无信用标签时不展示该行。
- 简介使用正文展示；缺失时展示“暂无简介”。

这个模块用于回答买家首要问题：商家卖什么、是否可信、是否值得继续查看资源。

### 地址与图片

地址和商家图片作为辅助判断信息，放在商家画像之后：

- 有地址或地图定位时展示地址块。
- 有地图定位时保留“地图导航”按钮。
- 有商家图片时使用横向滚动图库展示。
- 无图片时不展示图库，避免空卡片占据首屏。

### 发布与权益

发布概况不再独立重复统计，只保留轻量提示，说明当前发布和成交反馈数据。

权益提示保留，但视觉权重降低，用于解释认证、运营推荐和置顶权益的作用边界，避免对曝光和交易做过度承诺。

### 发布记录

发布记录继续使用现有 `ResourceCard`：

- 有资源时展示资源列表。
- 无资源时展示“暂无公开资源”。
- 点击资源仍跳转资源详情。

### 底部联系栏

底部固定栏保持现有隐私保护行为：

- “复制微信”继续提示从资源详情页查看完整微信。
- “拨打电话”继续提示从资源详情页查看完整电话。

本次不直接在商家主页暴露完整联系方式。

## 数据和异常处理

- `merchant.mainCategories` 缺失时使用简介或默认文案兜底。
- `merchant.creditTags` 缺失或为空时隐藏信用标签行。
- `merchant.images` 缺失或为空时隐藏图片模块。
- `merchant.addressText` 和地图定位都缺失时隐藏地址模块。
- 关注状态接口失败时继续保留当前页面浏览能力。
- 资源列表接口使用现有失败行为，本次不新增错误 UI。

## 验证标准

满足以下条件即可验收：

- 商家主页首屏优先展示商家身份、认证/核实标签、主营摘要和统计数据。
- 主营品类、信用标签和简介合并到“商家画像”模块。
- 地址、地图导航、商家图片、发布记录按现有数据条件展示。
- 关注/取消关注、自有商家编辑、地图导航、资源跳转、底部联系提示保持可用。
- 页面缺少 LOGO、主营品类、简介、地址或图片时不出现空白卡片或横向溢出。
- `wxapp` 验证脚本和小程序构建通过。
