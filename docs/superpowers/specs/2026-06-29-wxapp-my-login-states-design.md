# wxapp 我的页登录态展示设计

## 目标

完善 `wxapp/pages/my/index.vue` 在未登录和已登录两种状态下的展示效果，让用户能清楚理解当前账号状态和可执行操作。

本次只做页面展示和入口可用性的整理，不新增商家绑定能力，不调整后端接口协议。

## 范围

本次覆盖：

- “我的”页顶部账号信息展示。
- 未登录状态的登录引导。
- 已登录状态的账号信息和手机号绑定入口。
- 常用功能入口在未登录、已登录状态下的展示和点击反馈。

本次不覆盖：

- 商家 ID 手动填写和保存身份。
- 商家资料、商家认证、我的权益余量、权益刷新、置顶券余量等依赖商家身份的能力。
- 新增退出登录能力。
- 后端 API、登录协议、短信协议变更。

## 状态规则

页面只按本地 `token` 判断是否已登录：

- 有 `token`：视为已登录。
- 无 `token`：视为未登录。

`merchantId` 暂不作为本次页面展示依据。即使本地存在旧的 `merchantId`，页面也不展示商家工作台、权益或商家认证相关模块。

## 未登录状态

未登录时页面重点是引导用户登录：

- 顶部展示游客头像、标题“未登录”和说明“登录后管理需求、收藏和发布记录”。
- 主按钮展示“微信登录”。
- 展示登录后可用能力说明，例如“同步收藏关注”“查看我的需求”“接收审核和联系消息”。
- 常用入口仍可见，但点击需要登录的入口时提示“请先登录”。

未登录状态不展示：

- 手机号绑定表单。
- 商家 ID、用户 ID 输入框。
- 我的权益卡片。
- 商家资料、商家认证等商家入口。

## 已登录状态

已登录时页面重点是账号中心和常用操作：

- 顶部展示账号头像、标题“我的账号”和说明“已登录，可管理需求、收藏和消息”。
- 展示用户 ID 的弱提示；没有用户 ID 时展示“账号信息已同步”。
- 展示手机号绑定模块，保留发送验证码和绑定手机号的现有流程。
- 展示常用功能入口：我的需求、收藏关注、消息、发布资源。

已登录状态不展示：

- 商家 ID、用户 ID 手动输入和保存身份按钮。
- 我的权益、去使用权益、商家资料、商家认证。

## 交互与异常

- 微信登录成功后保存 `token` 和后端返回的 `user.id`，页面立即切换为已登录状态。
- 登录失败时沿用现有中文错误提示。
- 发送短信验证码和绑定手机号沿用现有逻辑。
- 未登录点击需要登录的入口时，只做中文 toast 提示，不跳转到不可用页面。
- 发布资源入口可以保留跳转，但未登录时先提示登录，避免用户进入后才发现状态不完整。

## 视觉方向

视觉沿用当前 wxapp v3 风格：

- 浅灰页面背景和白色卡片。
- 主色继续使用 `#0f766e`。
- 卡片圆角保持紧凑，不新增复杂视觉装饰。
- 页面结构保持移动端纵向列表，减少输入框堆叠。

## 验证标准

满足以下条件即可验收：

- 未登录时不再展示商家 ID、用户 ID、权益余量、商家资料、商家认证。
- 未登录时微信登录按钮明显可见。
- 已登录后不再展示手动保存身份表单。
- 已登录后手机号绑定流程仍可操作。
- 常用入口文案清晰，未登录点击受限入口有中文提示。
- `wxapp` 现有验证脚本通过。
