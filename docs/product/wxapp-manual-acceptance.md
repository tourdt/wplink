# 微信小程序手工验收

## 前置条件

- 后端生产或验收环境 `/healthz`、`/readyz` 均返回 `ok`。
- 微信公众平台已配置 request、uploadFile、downloadFile 合法域名。
- `wxapp/dist/build/mp-weixin` 已用生产 API 域名构建。
- 微信开发者工具已导入项目，AppID 使用正式或体验版 AppID。

## 验收流程

1. 登录与商家身份
   - 打开“我的”页。
   - 完成微信登录和短信验证码绑定。
   - 填入一个可管理的商家 ID。
   - 预期：页面展示商家信息、认证状态和权益余量。

2. 发布、审核、搜索、联系、指标
   - 在“发布”页填写资源标题、品类、数量、价格、联系人和电话。
   - 提交后进入发布成功页。
   - 后台 `/admin/resources/pending` 审核通过该资源。
   - 小程序“搜索”页输入关键词，进入资源详情。
   - 点击电话、复制微信、商家主页入口。
   - 回到“我的发布”。
   - 预期：资源状态为已发布，详情浏览、电话、微信指标增加。

3. 搜索无结果到采购需求
   - 搜索一个不存在的关键词。
   - 点击“提交需求”。
   - 填写需求标题、品类、数量和联系方式。
   - 进入“我的需求”。
   - 预期：需求列表出现新需求，后台采购需求页可见。

4. 认证和权益
   - 进入“商家认证”。
   - 提交主体名称、执照图片 URL、门头图片 URL。
   - 后台认证审核通过。
   - 回到商家主页和“我的”页。
   - 预期：展示认证状态、信用标签，权益余量更新。

5. 我的发布管理
   - 在“我的发布”按状态筛选。
   - 对已发布资源执行刷新、置顶、标记成交、下架。
   - 对已过期或已成交资源执行“再发类似”。
   - 预期：状态流转正确，草稿可再次提交审核。

6. 消息中心
   - 触发审核通过、驳回、认证结果或资源过期消息。
   - 进入消息中心，切换全部、未读、已读。
   - 点击未读消息。
   - 预期：消息可读，已读状态更新。

7. Banner、专题和 web-view
   - 后台新增或启用 Banner。
   - 小程序首页点击 Banner。
   - 分别验证跳转专题、资源详情、商家主页和白名单 web-view。
   - 预期：白名单链接可打开，非白名单链接显示不可访问。

8. 图片比例和裁切
   - 首页 Banner、专题 Banner 使用统一横图比例。
   - 首页活动卡、资源列表缩略图、同类推荐缩略图使用方图。
   - 资源详情主图使用 4:3，详情小图使用方图。
   - web-view 活动页封面使用 16:9。
   - 预期：图片使用中心裁切，不出现卡片高度跳动、图片挤压变形或页面横向滚动。

## 通过标准

- 以上核心流程全部通过。
- 无接口返回 `API_NOT_CONNECTED`。
- 无页面停留在永久 loading。
- 用户可见错误为中文且可操作。
- 图片上传、短信验证码、微信登录均使用真实生产服务。
