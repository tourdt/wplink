# 服链通 MVP 验收清单

## 1. 发布、审核、搜索、联系、指标

- 商家在小程序“我的”页保存商家 ID。
- 商家进入“发布”，填写资源标题、品类、数量、价格、联系人和电话。
- 提交后进入发布成功页，可跳转消息中心或我的发布。
- 运营在后台“资源审核”看到待审核资源。
- 运营通过审核后，资源进入公开搜索列表。
- 买家在小程序“搜索”输入关键词或选择资源类型，能看到资源卡片。
- 买家进入资源详情，能查看价格、数量、描述、商家认证状态。
- 买家点击复制微信、联系商家或进入商家主页，会触发联系行为记录。
- 商家在“我的发布”看到曝光、浏览、电话、微信等效果数据。

## 2. 搜索无结果到采购需求

- 买家搜索不存在的关键词。
- 页面展示“暂未找到合适资源”空态。
- 买家点击“提交需求”，填写需求标题、品类、数量和联系方式。
- 提交后进入需求成功页。
- 运营在后台“采购需求”看到该需求，并可标记跟进或关闭。

## 3. 认证和权益

- 商家进入“商家认证”，提交主体名称、执照 URL、门头 URL。
- 运营在后台“认证审核”看到待审核记录。
- 运营通过认证后，商家主页展示认证状态和信用标签。
- 后台“权益发放”可对商家发放发布额度或刷新额度，并要求填写原因。
- 操作日志中能看到认证审核和权益发放记录。

## 4. 我的发布和资源生命周期

- 商家在“我的发布”按状态筛选资源。
- 已发布资源可以刷新、置顶、标记成交、下架。
- 已过期或已成交资源可以“再发类似”生成草稿。
- 即将过期和已过期资源会产生消息提醒。
- 生命周期任务执行后，过期资源状态变更为 `expired`。

## 5. 消息中心

- 消息中心支持全部、未读、已读切换。
- 资源审核通过、驳回、下架、即将过期、已过期能产生消息。
- 认证结果能产生消息。
- 撮合进度能产生消息。
- 效果反馈能产生消息。
- 用户点击未读消息后，状态可更新为已读。

## 6. Banner、专题和 web-view

- 后台“Banner 专题”可新增或编辑 Banner 和专题配置。
- 首页能展示 active Banner。
- Banner 跳转专题时，能打开专题资源列表。
- Banner 跳转资源时，能打开资源详情。
- Banner 跳转商家时，能打开商家主页。
- Banner 跳转 web-view 时，先通过白名单校验，再打开活动页。
- 不在白名单的活动链接展示不可访问状态。

## 7. 人工撮合

- 运营在后台“人工撮合”基于采购需求创建撮合单。
- 撮合单可以追加候选资源和参与商家。
- 撮合状态支持 `open`、`contacted`、`succeeded`、`failed`、`closed`。
- 状态为 `succeeded` 或 `failed` 时必须填写结果说明。
- 撮合状态更新会写入操作日志。
- 撮合状态更新会向参与商家或用户发送站内消息。

## 8. 数据和审计

- `backend/scripts/seed_demo_data.sql` 可在干净数据库上重复执行。
- 后台“数据概览”展示待审核资源、待认证、采购需求和今日联系。
- 后台“操作日志”可按对象类型、对象 ID、操作者 ID 查询。
- 敏感操作至少覆盖审核通过、驳回、下架、发放、撤销、撮合状态更新。

## 当前验收限制

- 完整端到端手工点击验收依赖可连接的 PostgreSQL、已启动的后端 HTTP server、管理后台和微信开发者工具。
- 若当前执行环境缺少数据库连接或微信开发者工具，只能完成 SQL 静态校验、后端单元测试、后台构建和小程序构建；数据库可连接时应运行 `go run ./scripts/verify_migrations.go -config etc/app.yaml` 验证 migration up/down 和演示数据导入。
- 正式运营验收还应检查 `RuntimeMode: production` 下关键配置完整、微信 `jscode2session` 可用、短信验证码服务已接入、七牛图片上传可用，并确认微信公众平台已配置 request/uploadFile 合法域名。
